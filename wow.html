CMS theming

De Basis

  • HTML beschrijft de inhoud
  • CSS past de vormgeving toe
  • JavaScript zorgt voor dynamiek

JavaScript

  • JavaScript is vaak meer bijzaak bij theming.
  • JavaScript wordt vaak uitgebreid voor visuele gimmicks en speciale pagina's.
  • Bij theming kun je de CMS JavaScript meestal ongemoeid laten
    (alleen problemen bij heel drastische HTML aanpassing).
  • Het is wel heel belangrijk voor de standaard werking van Plone.

CSS

  • Voor theming maken HTML en CSS de dienst uit.
    CSS zet de vormgeving op een html pagina, bijvoorbeeld: Het toekennen van kleur, typography, achtergronden, schaduw, enz.
  • De structuur van de HTML heeft veel invloed op wat door CSS aan vormgeving mogelijk.
  • De CMS JavaScript gaat vaak uit van een bepaalde CSS voor het bewerkstelligen van de functionaliteit.
  • Vaak wordt een gedeelte van de Plone CSS als basis gebruikt om uit te breiden naar een nieuwe vormgeving.

HTML

  • De HTML voor CMS wordt opgebouwd door middel van veel kleine deelbestanden.
  • Theming heeft vaak tot gevolg dat deelbestanden moeten aangepast.
  • Hoe meer de vormgeving afwijkt van standaard Plone hoe meer deelbestanden moeten worden aangepast.
  • Vooral structurele aanpassingen in details heeft veel impact.
    (bijvoorbeeld andere html voor formulier knoppen, datum weergave, details die in veel templates gelijk moeten worden getrokken)

Gevolgen

  • Vormgeving met hoog gehalte aan kritieke details kan kostbaar worden.
  • The Devil is in the details!
    Veel templates aanpassen houd in dat er sneller één vergeten wordt.
  • Migraties worden ingewikkelder naar mate meer bestanden zijn aangepast.

Enter the WOW factor

Theming nieuwe stijl

Één Plone omgeving

                +

Één statische HTML blauwdruk

                +

Één bestand met vulregels

                =

Één Plone in vormgeving

Wat is zo bijzonder?

  • Met de HTML blauwdruk en vulregels kun je de volledige vormgeving van Plone aanpassen zonder één enkel Plone bestand te wijzingen!
  • Theme HTML structuur en Vormgeving is losgekoppeld van het CMS

Positieve bijkomstigheden

  • De content regels werken op basis van CSS kennis, waardoor Front-enders makkelijk instappen als Plone themer.
    (Plone ontwikkeling heeft een stijle leercurve)
  • Bij bestaande vormgeving kun je nu redelijk makkelijk Plone naar binnen schuiven als achterliggend CMS.
    (Bijvoorbeeld als je ben begonnen met een statisch HTML website, of als je met een niet meer ondersteund CMS zit, of je loopt tegen de grenzen van Wordpress aan)
  • Je kan een ontwerp bureau makkelijker HTML en CSS laten aanleveren, zonder dat je het weer helemaal moet herschrijven.
